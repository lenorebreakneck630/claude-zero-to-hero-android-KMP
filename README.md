## 🚫 Anti-patterns guide

The anti-patterns guide helps you spot code that gets hard to maintain.

It covers cases like:
- Putting too much logic in UI files
- Mixing network code with screen code
- Writing large classes that do too much
- Skipping tests on core flows
- Letting state spread across too many places
- Copying the same logic into several files

Use this guide when code feels hard to follow or hard to change.
